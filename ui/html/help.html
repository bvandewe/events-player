Help and Usage

Overview

Event Subscriber (handle events)

This micro-app serves as a test subscriber for CloudEvents: it displays CloudEvents on the UI as they are received on its POST /events/pub endpoint. Upon reception of a valid CloudEvent, the app distributes it to all browsers that are connected at that time, if any. It can also generate test events and send them to the selected CloudEvent Gateway. (Gateways can be added by Administrators with an Environment variable).

As a quick test: open the generator, select the localhost Gateway, and send an event. It will appear in the UI.

Event Publisher (emit events)

The generator sends customized events to the selected Gateway. When clicking the "Send" button, the backend app starts a background task that generates the event with the selected parameters, including Source, Type, Subject and Data. The Iteration parameter defines how many events are sent sequentially and the Delay parameter defines how long the backend waits before running the next iteration.

Connection Status

  • The connection to the backend is established and refreshed every 15s.
  • An event was received within the last 10s.
  • The connection to the backend is broken. Trying to re-establish it every 2s.

Keyboard Shortcuts

  • CTRL or META+UpArrowOpen the Generator
  • Escape or META+UpDownClose the Generator
  • META+fFocus/Unfocus on the Search/Filter box
